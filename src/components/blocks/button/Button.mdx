</Description>

## When To Use
- Primary and secondary actions that move users forward.
- Inline actions that should read like links in paragraphs.
- Confirmation actions where a loading state prevents double-submit.

## Do
- Use one primary button per view to keep focus clear.
- Set `loading` for async actions and provide `loadingText` for announcements.
- Use `href` for simple links or `asChild` for custom link components.

## Don't
- Stack multiple primary buttons with equal visual weight.
- Use glow on destructive actions or low-priority controls.
- Remove labels during loading when the action still needs context.

## Accessibility Notes
- Defaults to `type="button"` when rendered as a button element.
- `loading` sets `aria-busy` and disables interaction.
- `loadingText` is announced to assistive technologies.
- When rendered as a link, `aria-disabled` is applied and focus is removed while disabled/loading.

## Unity Glow
- Reserve glow for a single high-priority action in a view.
- Prefer `glowIntensity="soft"` for regular dashboards.
- Use medium or strong glow sparingly for hero moments.
